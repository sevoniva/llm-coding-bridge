# v0.7.0

`v0.7.0` makes the bridge a reliable multi-model local routing layer for Codex, Claude-compatible clients, ZCode, and OpenAI-compatible clients.

## Highlights

- Version 2 configuration separates client aliases, upstream model IDs, providers, and independent credential references.
- The guided `setup` flow can configure multiple models, probe them, install the local service, and safely manage one bridge-owned ZCode 3.x provider.
- Bounded retries, phase-aware deadlines, per-route cooldown, downstream heartbeats, and semantic-output tracking improve long-running request recovery without replaying visible model output.
- Chat, Responses, and Anthropic-compatible paths support shared routing and protocol-specific streaming behavior.
- The local `/admin` console shows redacted route health and bounded request timelines.

## Compatibility Fixes

- Empty, whitespace-only, `null`, and empty-array assistant history entries are removed before Chat Completions requests reach strict upstreams. Assistant entries containing tool calls, function calls, reasoning, refusal, or audio are preserved without an empty `content` field.
- Valid non-SSE Chat completions are normalized when a streaming client expects SSE.
- Version 1 configuration remains supported. Use `llm-coding-bridge config migrate --dry-run` before opting into version 2.

## Security And Operations

- Credentials stay outside the bridge config and can be isolated per model through environment, command-backed, or client-provided sources.
- Generated secret-bearing files and backups use private permissions; configuration updates are atomic and preserve supported symlinks.
- CI now tests Node.js 18, 20, 22, and 24. Publishing verifies the tag/version pair, repository and package boundaries, secrets, high-severity dependency audit, and npm provenance.

## Upgrade

```bash
npm install -g @sevoniva/llm-coding-bridge@0.7.0
llm-coding-bridge config migrate --dry-run
llm-coding-bridge setup
llm-coding-bridge doctor --all-models
llm-coding-bridge restart-service
```

Package installation does not rewrite local configuration. Preview ZCode changes with `llm-coding-bridge client add zcode --dry-run` before applying them.

## Validation Note

The automated fast gate, deterministic multi-model acceptance matrix, short fault-soak validation, local static scan, secret scan, dependency audit, repository gate, and package dry run passed before release. The full 20-minute soak, interactive Codex Security scan, and real-provider network/sleep acceptance were not executed for this expedited release and are recorded as validation gaps.
